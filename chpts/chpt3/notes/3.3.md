# 3.3 Asymptotic Analysis
- Growth rate of the running time as a function of the input size $n$

Example:
```python
def find_max(data):
    """Return the max elt from a nonempty python list"""
    biggest = data[0]

    for val in data:
        if val > biggest:
            biggest = val
    
    return biggest
```
This example is linear time `O(n)` since the loop does `len(data)` iterations where each iteration performs a constant time comparison operation of `O(1)`

## 3.3.1 The "Big-Oh" Notation
- **Big-O**:
    - A function is "less than eq to" another function
    - Let $f(n)$ and $g(n)$ be functions mapping positive integers to positive real numbers.
        - We say that $f(n)$ is $O(g(n))$ if there is a real constant $c > 0$ and an integer constant $n_0 \geq 1$ s.t. =>
            - $f(n) \leq cg(n)\text{, for } n \geq n_0$
        - This is also defined by $f(n)$ is big-O of $g(n)$

- **Big-Omega**:
    - A function is "greater than or eq to" another function
    - $f(n) \geq cg(n) \text{, for } n \geq n_0$
        - i.e. $3nlogn-2n$ is $\Omega(nlogn)$

- **Big-Theta**:
    - A function grows at the same rate as another function up to constant factors => 
        - $f(n)$ is $\Theta(g(n))$ if $f(n)$ is $O(g(n))$ and $f(n)$ is $\Omega(g(n))$
