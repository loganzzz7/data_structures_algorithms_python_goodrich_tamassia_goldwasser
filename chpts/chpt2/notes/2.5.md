# 2.5 Namespace and Object-Orientation
- A namespace is an abstraction that manages all of the identifiers that are defined in a particular scope -> mapping each name to its associated value
    - functions, classes and modules are all first-class objects so the value associated with an identifier in a namespace may be a function, class or module

## 2.5.1 Instance and Class Namespaces
- **Instance namespace** -> manages attributes specific to an individual object
    - each instance of our CreditCard class maintains a distinct balance, a distinct account number, a distinct credit limit ...
        - Therefore, each credit card has a dedicated instance namespace to manage such values
- **Class namespace** -> a separate namespace for each class that has been defined
    - the namespace is used to manage members that are shared by all instances of a class
        - i.e. the `make_payment` method of the CreditCard class -> class functions or `GLOBAL_VARS`--defined by caps i.e. `OVERLIMIT_FEE = 5`
- **Nested classes** -> nest one class definition within the scope of another class
    - Nesting one class in the scope of another makes claer that the nested class exists for the support of the outer class
        - this \neq inheritance--the nested class does not inherit from the outer class
    - **This is useful when** implementing `linked list`, where a linked list class will have an inner `node` class which will be used to store each node in the list

```python
class A: # outer class
    class B: # nested class
        # ...
```
- **Dictionaries and the `__slots__` declaration** -> python represents each namespace with an instance of the built-in `dict` class that makes identifying names in that scope to the associated objexts
    - to streamline the representation of all instances of a class -> provide a class-level member named `__slots__` that is assigned to a fixed sequence of strings that serve as names for instance variables
        - i.e. in the `CreditCard` class -> `__slots__ = '_customer', '_bank', '_account', '_balance', '_limit'`
    - a subclass of the base class would also declare `__slots__` to acoid creation of instance dictionaries -> a subclass should **only** have names of supplemental methods that are newly introduced
        - i.e. in the `PredatoryCreditCard` subclass -> `__slots__ = '_apr' # inaddition to the inherited mambers`
    - This `__slots__` method can be used to streamline every class

## 2.5.2 Name Resolution and Dynamic Dispatch
- Python uses `Dynamic Dispatch` to determine at runtime which implementation of a function to call based on the type of object upon which it is invoked
    - sub-class methods **OVERRIDES** the base class methods of the same name!
