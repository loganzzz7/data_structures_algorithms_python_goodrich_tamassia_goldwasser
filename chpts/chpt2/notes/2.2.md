# 2.2 Software Development
- Software Development Phases (Three major steps):
    - **Design**
    - **Implementation**
    - **Testing and Debugging**
## 2.2.1 Design
- Rules of Thumb for designing our classes:
    - **Responsibility**:
        - Divide the work into different **actors**, each with a different responsibility
            - Describe responsibilities with action verbs!
    - **Independence**:
        - Define the work for each class to be as independnet from other classes as possible
            - each class should have autonomy over some aspect of the program, give data to the class that has jurisdiction over the actions that require access to this data
    - **Behaviors**:
        - Define behaviours for each class carefully -> the consequences of each action performed by a class will be well understood by other classes that interact with it
            - **Interface**: the set of behaviours that are defined for a class
- **CRC cards**: use Class-Responsibility-Collaborator cards when designing
    - these should define each class, the work done by the class, and the other classes that would work with this class

Example Class Diagram:
```
class (name): CreditCard
Fields (instance variables): _customer, _bank, _account, _balance, _limit
Behaviors (methods): get_customer(), get_bank(), get_account(), make_payment(amount), get_balance(), get_limit(), charge(price)
```

### 2.2.4 Testing and Debugging
- **Testing**:
    - for dependencies across classes -> test **top-down** or **bottom-up**
    - use **stubbing** to stub out a part of the code to test around it and isolate the error

Can write a `main()` to test an individual module, invoked by:
```python
if __name__ == '__main__':
    # perform module specific tests
```
- **Debugging**:
    - use `print()` stmts to test
