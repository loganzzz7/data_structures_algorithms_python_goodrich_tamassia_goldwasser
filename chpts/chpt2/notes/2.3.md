# 2.3 Class Definitions
- A class: primary means for abstraction in oop
    - every piece of data is represented as an instance of some class
    - a class provides a set of behaviours in the form of member functions--aka methods

Example: CreditCard Class
- construct begins with keyword `class`
    - the `self` identifier:
        - `self` identifies the instance upon which a method is invoked -> each self maintains their own instance variables

Code:
```python
class CreditCard:
    def __init__(self, customer, bank, acnt, limit):
        """
        create a new credit card instance

        init balance = zero

        customer -> name of customer
        bank -> name of the bank
        acnt -> the account identifier
        limit -> credit limit
        """
        self._customer = customer
        self._bank = bank
        self._account = acnt
        self._limit = limit
        self._balance = 0

    def get_customer(self):
        """return name of the customer"""
        return self._customer
    
    def get_bank(self):
        """return the bank's name"""
        return self._bank
    
    def get_account(self):
        """return the card identifying number (typically stored as a string)"""
        return self._account
    
    def get_limit(self):
        """return the current credit limit"""
        return self._limit
    
    def get_balance(self):
        """return the current balance"""
        return self._balance

    def charge(self, price):
        """
        charge given price to card
        
        return True if charge was processed
        False if declined
        """

        if price + self._balance > self._limit: #charge exceeds so false
            return False
        else:
            # else went thru so make charge & true
            self._balance += price
            return True
        
    def make_payment(self, amount):
        """
        process customer payment that reduces balance
        """
        self._balance -= amount
```

To create an instance--`constructor`:\
`cc = CreditCard('John Doe', '1st bank', '5391 0375 9387 5309', 1000)`

Table of overloaded operations
```
common syntax -> special method form
a + b -> a.__add__(b) alternatively b.__radd__(a)
a - b -> a.__sub__(b) ...
...
```

- **Implied Methods**:
    - if a particular special method is not implemented in a user-defined class -> standard syntax that relies upon that method will raise an exception
    - i.e. a + b if a is an user defined class without an `__add__` method then an error will be raised
