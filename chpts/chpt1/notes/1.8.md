# 1.8 Iterators and Generators
- **ITERATOR**: an obj that manages an iteration through a series of values
- **ITERABLE**: an obj that produces an iterator via the syntax iter(obj)
```
given: data = [1, 2, 3, 4]
cannot: next(data)
need to: i = iter(data)--this is an iterator object!; next(i)
```
- **GENERATORS**: technique for creating iterators in Python
    - implemented with a syntax that is similar to a function but instead of returning values, a yield stmt is executed to indicate each elt of the series.
        - i.e. number 100 has factors 1, 2, 4, 5, 10, 20, 25, 50, 100
Traditional function to produce and return a list containing all factors:
```python
def factors(n):
    res = []
    for k in range(1, n + 1):
        if n % k == 0:
            res.append(k)
    return res
```
A generator:
```python
def factors(n):
    for k in range(1, n + 1):
        if n % k == 0:
            yield k # yield this factor as next res
```
Then can do `for factor in factors(100):` -> an instance of the generator is created and for each iteration of the loop, executes procedure until a yield stmt indicates the next val.
- fib seq example:
```python
def fib():
    a = 0
    b = 1

    while True:
        yield a
        future = a + b
        a = b
        b = future
```
