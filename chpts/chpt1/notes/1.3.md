# 1.3 Expressions, Operators, Precedence
- Logical operators:
    - not: unary negation
    - and: conditional and
    - or: conditional or
- Equality Operators:
    - is: same identity
    - is not: different identity
    - ==: equivalent
    - !=: not equivalent
        - `==` is a more general notion of equivalence -> `a == b` will be True if `a` and `b` referred to the same object or if the values of `a` and `b` are the same where as `a is b` will be True iff `a` and `b` refer to the same object.
- Comparison Operators:
    - <: less than
    - <=: leq
    - <: gtr than
    - `>=`: geq
- Arithmetic Operators:
    - +: addition
    - -: subtraction
    - *: mult
    - /: true division
    - //: integer division -> returns an int that is the **FLOOR**
    - %: modulo
- Bitwise:
    - ~: complement
    - &: and
    - |: or
    - ^: xor--returns 1 only if two differ
    - <<: left shift
    - `>>`: arithmetic right shift -> fills w sign-bit!
- Sequence:
    - s[j]: elt at idx j
    - s[start:end]: slice from start to end--excluding end
    - s[start:end:step]: slice from start to end skipping via step
    - s + t: concat two sequences
    - k * s: s + s + s... (k times)
    - val in s: True if val is contained in s
    - val not in s: True if val is not in s
    - s == t: equiv (elt by elt)
    - s != t: not equiv
    - s < t: lexicographically less than
    - `s > t`: lexicographically gtr then
    - s <= t: lexicographically leq
    - `s >= t`: lexicographically geq
- Sets and Dicts:
    - key in s: -> check if key in s
    - key not in s:...
    - s1 <= s2: s1 is subset of s2
    - s1 < s2: s1 is proper subset of s2
    - s1 >= s2: s1 is superset of s2
    - s1 > s2: s1 is proper superset of s2
    - s1 | s2: union of s1 and s2 -> s1 or s2
    - s1 & s2: intersection of s1 and s2 -> s1 and s2
    - s1 - s2: elts in s1 but not s2
    - s1 ^ s2: elts in precisely either s1 or s2 -> s1 xor s2!
    - d[key]: value associated with given key
    - d[key] = value: set the value associated with the key
    - del d[key]: delete key and associated val from dict
    - key in d: containment check
    - key not in d: non-containment check
    - d1 == d2: dict1 eq dict2
    - d1 != d2: dict1 neq dict2
- Extended Assignment Operators
    - i.e. `+=, -=, ...`
## 1.3.1 Compound Expressions and Operator Precedence:
- Operator precedence table:
```
1 member access -> expr.member
2 func calls & container slices -> expr() & expr[s:e]
3 exp -> **
4 unary ops -> +expr, -expr, ~expr
5 mult & div -> *, /, //, %
6 add, sub -> +, -
7 bit-wise shift -> <<, >>
8 bit-wise and -> &
9 bit-wise xor -> ^
10 bit-wise or -> |
11 comparisons & containments
12 logical not
13 logical and
14 logical or
15 conditional -> val1 if cond else val2
16 assignments -> =, +=, -=, *=, ...
```
