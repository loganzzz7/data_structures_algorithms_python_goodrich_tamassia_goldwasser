# 1.2 Objects in python
- Python is an object-oriented language and classes form the basis for all data types.
    - Python's built-in classes: int--integers, float--floating pt values, str--char strings
## 1.2.1 Identifiers, Objects, and the Assignment Statement
- `temperature = 98.6`
    - temperature = identifier
    - 98.6 = floating point object
    - setup: `identifier = object`
- identifiers in python are case sensitive -> `Temperature != temperature`
    - identifiers **CANNOT** begin with a **NUMBER**
        - i.e. `9lives` == illegal
- There are 33 reserved words that cannot be used as identifiers
```
False, as, continue, else, from, in, not, return, yield, None, assert, def, except, global, is, or, try, True, break, defl, finally, if, lambda, pass, while, and, class, elif, for, import, nonlocal, raise, with
```
- Python identifers ~= pointer variable in c++
    - Each identifier is implicitly associated with the memory address of the onject which it refers -> all vars in python are pointers!!
    - Assigning to None == NULL reference in c++
- Python is a dynamically typed language -> no advance declaration associating an identifier with a data type 
    - an identifier can be associated with **ANY** type of object
    - it can also be reassigned to another object of the same **(or different)** type
- an alias can be extablished by assigning a second identifier to an existing object. 
    - `original = temperature` -> now original is a second identifier referencing the original existing object
        - `temperature` and `original` are aliases for the same object
- if one of those names is reassigned o a newvalue via a subsequent assignment stmt -> the aliased obj does not get affected and it instead breaks the alias!!
    - `temperature = temperature + 5.0`
        - now `temperature = float obj of 103.6`
        - and `original = float obj of 98.6`
    - RHS is evaluated first--`temperature +5.0`
        - the result is stred as a new floating-point instance which then is assigned to the LHS indicator--`temperature`
## 1.2.2 Creating and Using Objects
- Instantiation: process of creating a new instance of a class
    - invoke the constructor of a class
        - i.e. if there \exists a class named Widget then instantiate with `w = Widget()`
    - python's built-in classes support **literal** form for designing new instances.
        - i.e. `temperature = 98.6` => creation of a new instance of the float class--98.6 = literal form.
- Calling Methods: `name.func()` -> expression to the left of the dot identifies the obj upon which the method is invoked.
    - This can also be piped -> `response.lower().startswith('y')` -> first evals `response.lower()` which returns a new str instance then does `.startswith('y')` method on the intermediate str
    - **accessors**: methods that do not change the state of an object and rather returns info about the state of an obj
    - **mutators or update methods**: changes the state of an obj -> i.e. `placeholderlist.sort()`
## 1.2.3 Python's Built-In Classes
- Table of the commonly used, built-in classes in Python
    - a class is immutable if each obj of that class has a fixed value upon instantiation that cannot be subsequently changed -> i.e. float class = immutable, once it is created, the value cannot be changed
```
bool immutable
int immutable
float immutable
list mutable
tuple immutable
str immutable
set mutable
frozenset immutable
dict mutable
```
- Bool class: used to manipulate logical vals -> True / False
    - default constructor `bool()` returns False!!
    - can be used to evaluate a non bool val via `bool(foo)`
        - False if numbers == 0 else True
        - False if len(iterable) == 0 else True
- Int class: designed to represent int values with arbitrary magnitude
    - python auto chooses the internal representation for an integer based on the magnitube of its value -> rather than having to declare (int, short, long) in c++
    - can also represent (binary, octal, hex):
        - `0b.... 0o.... 0x....`
    - `int(float)` truncates the float 
        - i.e. `int(3.14) == 3`
- Float class: fixed-precision representation -> more similar to a `double` in c++
    - can use scientific notation!! `3.14e16 = 3.14 * 10^{16}`
- Sequence Types: List, Tuple, Str
    - List class: a referential structure -> stores a sequence of references to its elements
        - array-based & zero-indexed
    - Tuple class: immutable version of a sequence
        - delimited by ()
    - Str class: designed to represent an immutable sequence of chars based on the unicode character set
        - `\`: escape character
        - `''' or """`: allow for newline chars to be embedded naturally -> no need for `\n` and can srite multiline strings in code
    - Set and Frozenset classes: set is implemented based on a **hash table** -> so set provides quick search `O(1)`!
        - Set: only instances of immutable types can be added to a Python set
        - Frozenset: a set that is immutable -> therefore -> legal to have a set of frozensets!
    - Dict class: dict represents a dictionary / mapping -> distinct keys to associated vals
        - delimited by {}
        - also accepts: `dict(pairs)` where `pairs = [('ga', 'Irish'), ('de': 'German')]`
