# 1.7 Exception Handling
- Exceptions: Unexpected events that occur during the execution of a program
    - Might result from: logical error or an unanticipated situation.
        - In Python, **EXCEPTIONS ARE OBJECTS** that are raised by code that encounters an unexpected circumstance.
        - The interpreter can also raise an exception shold it encounter an unexpected condition--i.e. out of memory error
        - A raised error can then be caught with a surrounding context that "handles" the exception
            - if uncaught -> can cause interpreter to stop executing and report an appropriate msg to the console
- Common Exception Types:
    - Exception class below:
```
class -> desc:
Exception -> base class for most error types
AttributeError -> Raised by suntax obj.foo, if obj has no memver named foo
EOFError -> Raised if end of file reached for console or input file
IOError -> Raised upon failure of I/O op--i.e. opening file
IndexError -> Raised on out of bounds index
KeyError -> Raised if nonexistent key requested for set or dict
KeyboardInterrupt -> Raised by ctrl-c cmd from user that stops program from executing
NameError -> Raised if nonexistent identifier used
StopIteration -> Raised by next(iterator) if no element
TypeError -> Raised when wrong type of param is sent to a func
ValueError -> Raised when param has invalid val
ZeroDivisionError -> Raised when any division operator used with 0 as divisor (div by 0 error)
```
### 1.7.1 Raising an Exception
- An exception is thrown by the **raise** stmt.
    - i.e. if a func for computing a sq root is sent a neg val as a param, it can raise an exception with cmd:
        - `raise ValueError('x cannot be negative')`
