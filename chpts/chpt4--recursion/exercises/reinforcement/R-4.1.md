## Describe a recursive algorithm for finding the maximum element in a sequence, $S$, of $n$ elements. 
```Python
def find_max_elt(S, l, r):
    if l == r:
        return S[l]
    
    mid = (l + r) // 2
    left_max = find_max_elt(S, l, mid)
    right_max = find_max_elt(S, mid, r)
    
    return max(left_max, right_max)
```

## What is your running time and space usage?
- Running Time: $T(n) = 2T(n/2) + O(1)$ -> O(n)
    - There are two subproblems of size $n/2$ and in each iteration there is an $O(1)$ operation of `max()`
- Space Usage: $O(logn)$
    - The space usage is $logn$ because the maximum number of activation frames is half the problem size since the subproblems have size $n/2$ so $logn$
