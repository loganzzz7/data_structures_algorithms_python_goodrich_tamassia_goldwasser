# 4.4 Further Examples of Recursion
- Additional examples of recursion -> organized by the maximum number of recursive calls that may be started from within the body of a single activation; below lists the different types of recursion:
    - If a recursive call starts at most one other, this is a **linear recursion**
    - If a recursive call may start two others, this is a **binary recursion**
    - If a recursive call may start three or more others, this is a **multiple recursion**

## 4.4.1 Linear Recursion
- If a recursive function is designed so that each invocation of the body makes at most one new recursive call, this is known as **linear recursion**
    - i.e. `good_fib`, `binary_search`
        - In each recursive branch -> one additional recursive call!
        - This reflects the structure of the recursion trace **NOT** the asymptotic runtime--`binary_search` is $O(logn)$
    - **Summing the Elements of a Sequence Recursively**:
        - Linear recursion can be a useful tool for processing a data sequence, such as a python list
        - Suppose we want to compute the sum of a sequence $S$ of $n$ ints
            - Solve this summation problem using linear recursion by observing that the sum of all $n$ integers in $S$ is trivially 0 -> so if $n = 0$ ret 0 else it is the sum of the first $n - 1$ integers in $S$ plus the last elt in $S$
            ```Python
            def linear_sum(S, n):
                """ret the sum of the first n numbers of sequence S"""
                if n == 0:
                    return 0
                else:
                    # this recursively solves the summation up to the last elt then add the last elt in S after recursive calls finish!
                    return linear_sum(S, n - 1) + S[n - 1]
            ```
    - **Reversing a Sequence with Recursion**:
        - Want to have the first elt become the last, the second elt become the second to last ...
            - Solution with **linear recursion**:
            ```Python
            def reverse(S, start, stop):
                """reverse elts in implicit slice S[start:stop]"""
                if start < stop - 1:
                    # base case where there are only two elts left
                    # -> do reversal
                    S[start], S[stop - 1] = S[stop - 1], S[start]
                    # recursive calls on the rest!
                    reverse(S, start + 1, stop - 1)
            ```
            - Note:
                - There are two implicit base cases:
                    1. When `start == stop` -> the implicit range is empty
                    2. When `start == stop -1` -> the implicit range has only one elt
                - In either of these cases, there is no need to do anything since a sequence with 0 or 1 elt is trivally eq to the reversal!
            - The code snippet completes in $1 - \lfloor \frac{n}{2} \rfloor$ recursive calls -> since each time there are two elts swapped
                - This runs in $O(n)$ time
    - **Recursive Algorithms for Computing Powers**:
        - As another interesting example of the use of linear recursion -> consider the problem of raising a number $x$ to an arbitrary non-negative int $n$ 
        - power function: $power(x, n) = x^n$
            - base case: n = 0 -> $x^0 = 1$
            - recursive case: $x * power(x, n - 1)$
        ```Python
        def power(x, n):
            """compute the value of x^n for integer n"""
            if n == 0:
                # base case
                return 1
            else:
                return x * power(x, n - 1)
        ```
        - This has a runtime of $O(n)$
    - **Improved Power Function--Squaring**:
        - Let $k = \lfloor \frac{n}{2} \rfloor$ denote the floor of the division--`n // 2`
        - Consider $(x^k)^2$ ->
            - When $n$ is even: $\lfloor \frac{n}{2} \rfloor = \frac{n}{2}$ -> $(x^k)^2 = (x^{\frac{n}{2}})^2 = x^n$
            - When $n$ is odd: $\lfloor \frac{n}{2} \rfloor = \frac{n - 1}{2}$ and $(x^k)^2 = x^{n - 1}$ -> therefore $x^n = x * (x^k)^2$ just as $2^13 = 2 * 2^12 = 2 * 2^6 * 2^6$
                - Constructed recursive definition:
                    1. if $n = 0$ -> 1
                    2. if $n > 0$ && is odd -> $x * (power(x, \frac{n}{2}))^2$
                    3. if $n > 0$ && is even -> $(power(x, \frac{n}{2}))^2$
        - If implemented this recursion making two recursive calls to compute $power(x, \frac{n}{2}) * power(x, \frac{n}{2})$ -> runtime = $O(n)$
            - Possible to perform fewer operations by computing $power(x, \lfloor \frac{n}{2} \rfloor)$ as a partial result then multiplying it by itself.
            ```Python
            def power(x, n):
                """compute the value x^n for int n"""
                if n == 0:
                    return 1
                else:
                    partial = power(x, n//2)
                    if n %2 == 1:
                        # odd case
                        return x * partial * partial
                    else:
                        # even case
                        return partial * partial
            ```
            - By splitting cases, similar to `binary_search`, the runtime of this improved algorithm is $O(logn)$ as there are only $logn$ calls as opposed to the previous algo with $n$ calls!
                - Since the recursive depth is also $logn$ as there are only $logn$ activation records -> the space complexity is also $O(logn)$

## 4.4.2 Binary Recursion
- When a function makes 2 recursive calls -> `binary recursion`
    - i.e. `draw_ruler`, `bad_fib`, 
        - `sum_n_elts_of_S` -> recursively compute the sum of the first half, the sum of the second half and add the sums together
        ```Python
        def binary_sum(S, start, stop):
            """return the sum ofthe numbers in implicit slice S[start:stop]"""
            if start >= stop:
                # trivial case 1: zero elts in slice
                return 0
            elif start == stop - 1:
                # trivial case 2: one elt in slice
                return S[start]
            else:
                # non trivial need to compute
                mid = (start + stop) // 2
                return binary_sum(S, start, mid) + binary_sum(S, mid, stop)
        ```
        - This version of sum of all elts has space $O(logn)$ since range is divided in half at each recursive call and the depth of trace stack is $logn$ however, the running time is still $O(n)$ since there are $2n - 1$ function calls -> each call besides base case has 2 so $2n$ and base case is $-1$

## 4.4.3 Multiple Recursion
- **Multiple Recursion**:
    - a process in which a function may make more than two recursive calls
        - i.e. analyzing the disk space usage of a file system -> because the number of recursive calls made during one invocation is random based on the number of entries within a given child directory
        - Another common application of multiple recursion is when we want to enumerate various configurations in order to solve a combinatorial puzzle! -> summation puzzle:
            ```
            pot + pan = bib
            dot + cat = pig
            boy + girl = baby
            ```
        - To solve such a puzzle -> need to assign a unique digit--0, 1, ..., 9 to each letter in the equation in order to make the equation valid
            - If the number of possible configurations is not too large -> use a computer to enumerate all the possibilities and test each one!
            - An algorithm can use **multiple recursion** to work through the configurations in a systematic way
                - The algorithm enumerates and tests all `k-length` sequences without repetitions of the elts of a given universe $U$ -> build the sequence of $k$ elts via the following:
                    1. recursively generating the sequences of $k - 1$ elts
                    2. appending to each such sequence an elt not alr contained in it
                - Use set $U$ to track the elts not contained in the current sequence, so that an elt $e$ has not been used yet iff $e$ is in $U$
                ```Python
                Algorithm PuzzleSolve(k, S, U):
                    """
                    Input: an integer k, sequence S, set U
                    Output: an enumeration of all k-length extensions to S using elts in U without repetitions
                    """
                    for each e in U do:
                        add e to the end of S # e is now being used so remove from U
                        U.remove(e)
                        if k == 1 then:
                            test whether S is a configuration that solves the puzzle
                            if S solves the puzzle then:
                                return "Solution found: " S
                        else:
                            # recursive call with trying next int
                            PuzzleSolve(k - 1, S, U)
                        # e didnt work so remove e from answer S and add it back to unused set U
                        S.remove(e)
                        U.add(e)
                ```
